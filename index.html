Devices

In 2010 I started working on Kobo's first E-Ink device. Since then, I've helped create 8 eReaders which got faster and prettier while winning industry awards.

E-Ink readers are devices designed to mimic the experience of reading on a regular piece of paper. They have high res, low-glare screens which last for months before needing to be charged and can carry a user's entire book library.

In 2010 The Kobo was born.
It had a big blue button and would sync your library via bluetooth.

The next generation was the Kobo Wifi. You could now sync directly to the cloud!

The Kobo Touch used a touchscreen and won awards.

Our next device, the Kobo Glo, introduced a front light

"One of the fastest, most responsive e-readers we've used."
PC World

Next we went small with the Kobo Mini

"The Mini is probably lighter than this week's issue of The New Yorker."
Huffington Post

And then we went big with the Kobo Aura HD

"the Porsche of eReaders."
PCMag.com
"Kobo Aura is the best flagship eReader on the market."
Sept 5, 2013 Engadget

The devices are built in C++ on top of the Qt framework and they run Linux. Having been on the project from the very beginning gave me an opportunity to work on pretty much every aspect of it. In particular, I've spent a bunch of time working on the persistance layer (including a custom cache) and the syncing component. I've also helped improve the speed and reliability of opening books and wifi connectivity.

Websites

My first experience with websites, was rewritting the reporting framework for the Tivoli Provisioning Manager from scratch. The project made heavy use of JSPs running on top of WebSphere and backed by DB2.

In my next project, at Marsh Canada, I dove deep into the UI side. I built and extended javascript pages and components, based on YUI and Ajax. In the process I gained a new appreciation for how beautiful javascript code can actually be.

  • Javascript & YUI
  • Java
  • JSP
  • WebSphere
  • DB2

At Kobo, I was asked to join a SWAT team focused on improving the performance of the kobobooks.com website, in order to handle expected huge spikes in traffic. We transitioned to a service oriented architecture and started using Membase (now Couchbase) while also doing various refactoring work to improve performance.

  • .NET
  • IIS
  • Couchbase
  • MS SQL

I continue occasionally building websites on the side, to try to stay on top of emerging trends and technologies. One recent example is "idealist". It's a website internal to Kobo built using django, running on apache and backed by postgresql. Essentially it's an image-heavy brainstorming tool that allows employees to submit cool ideas and features we should build.

  • Django
  • Python
  • Apache
  • Postgresql

Desktop Applications

Content is the lifeblood of an eBook company. At Kobo I built the original content processing system. It took books from publishers in various formats, parsed and converted them into a proprietary format to be consumed by our clients. The system was built in .NET and backed by MS SQL.

  • .NET
  • Windows
  • MS SQL

Later on while building the eReader devices, my team created the Kobo desktop application. Developed in Linux, it's built with C++ using Qt and runs on Windows as well as OS X.

  • C++
  • Qt
  • Windows/Linux/OS X
  • SQLite
